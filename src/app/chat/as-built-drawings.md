## 📋 Questions to Define Your As-Built Drawings Document

To create a comprehensive and accurate As-Built Drawings Document that meets your project's specific needs, please provide details on the following:

### 1. Project Overview
- What is the project name and location?
- What type of construction project is this (residential, commercial, industrial, infrastructure)?
- What is the current status of the project (ongoing, recently completed, or planned)?

### 2. As-Built Documentation Scope
- Which drawings need to be converted to as-built format (architectural, structural, MEP, civil, or all disciplines)?
- Are there specific systems or areas that require detailed as-built documentation (foundation, framing, electrical, plumbing, HVAC)?
- Do you have existing record drawings, shop drawings, or design drawings to work from?

### 3. Construction Modifications and Changes
- Were there any significant design changes during construction that need to be documented?
- Are there any change orders, field modifications, or value engineering decisions that affected the final build?
- What deviation exists between the original design documents and the actual constructed conditions?

### 4. Stakeholder Requirements
- Who will be the primary users of this As-Built Drawings Document (facility managers, future contractors, architects, engineers)?
- Are there specific deliverable formats required (hard copies, PDF, CAD files, BIM models)?
- What level of detail is required for maintenance and future renovation purposes?

### 5. Technical Specifications and Standards
- Are there specific industry standards or codes that must be followed (ASTM, ANSI, local building codes)?
- Do you require integration with Building Information Modeling (BIM) systems?
- What coordinate system and measurement units should be used (metric, imperial)?
